Curiosity Music Guide
Music Player Information
Welcome to Curiosity Music !
First of it's good to understand that
the Queue and the Playlist are 2 complete
seperate things. The Queue are the songs you
and your fellow members want to play. The playlist
are the songs Curiosity will be playing after the
queue is finished. This can be toggle on and off
in the settings menu.
The Queue:
With the play command you can put songs queue.
Curiosity will play them one by one until it's empty
After this it will turn to the servers playlist
If the playlist is playing and you put a new song in
the queue. Curiosity will first play the songs in the queue
before switching back to the playlist.
The Playlist:
The first time Curiosity will play the playlist on
your server it will first inject all song to our database.
We have a pre-configured list (5000+) of songs ready to
go. You can completely modify your servers playlist if
you like to. With the save and delete command you can
add or delete song from it. You can also decide to make
a brand new server playlist. Then you can choose your
own songs to play after the queue is finished.
Saving and Deleting songs:
This can be done once a song is playing. These commands
do not take any input. You can right now not save or delete
a song that is not currently playing. If you want to save a
song, first put it in the queue. Then once it's playing
you can save it. The same counts for deleting a song.
In a later update we will make sure you can do this
in multiple ways.
DISCLAIMER
Currently this module is getting build on. Bugs can be
encountered. We are working hard on fixing these in the
future. Currently known bugs are the following.

* If a new server starts playing music the bot needs
to inject all the default playlists songs in the database.
This will result in all servers having laggy or no music
for a duration of 5 / 10 seconds. This bug can unfortunately
not be fixed but it also will not happen only once per server.

* Sometimes when the pause command is used Curiosity
will stop playing music but continues to transmit.
Because of this it does not register it is paused and
the resume will not work. to fix you need to let the
bot rejoin the voice channel.

* If music commands are used to fast after each other
the music can start to skip every 2 seconds. To prevent
this we have put in a 10 second cooldown between each
time a command is used. If it does occure letting the bot
rejoin the voice channel is the solution.
* Sometimes a song stops before it actually is finished.
This is an internal bug within the FFMPEG. This module
converts the videos to an mp3. In here it sometimes
go's wrong. I can not fix this but if i have a better
alternative i will switch to

CMD Input /
[Optional]		 Result
Delete Delete current playing song from your playlist
Join Let Curiousity join the current voice channel
Leave Let Curiosity leave the current voice channel
NP Shows you the information about the current playing song
Pause Pause the current playing song
Play Name / URL Place a song in queue. Supports multiple platforms
Resume Resumes the current paused song
Save Save the current song to your server playlist
Skip Skip the current playing song
Volume 1 / 100 Set the global music volume from 1 to 100
